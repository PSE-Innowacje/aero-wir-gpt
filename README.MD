## Commiting and executing tasks

When executing a task from sc-docs task, create a separate branch with name: <TASK-NAME>-<QUICK-SUMMARY> where
TASK-NAME - 1-SC-AERO
QUICK-SUMMARY - initial commit
example 1-SC-AERO-initial-commit

Then when commiting, include task name in the commit and short summary
for example:
1-SC-AERO implementing initial setup for the backend project

When task is finished, move it to separate folder in tasks (create if needed) called finished
