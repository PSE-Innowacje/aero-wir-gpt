Co AI robi dobrze


  • Prędkość — pełny backend + frontend w kilka dni, nie tygodni
  • Skupienie na MVP — AI rozumie priorytet biznesowy, nie grzęźnie w detalach
  • Zmiana stacku — migracja Couchbase → MongoDB przeszła gładko, AI przepisał wszystko
  • Scaffolding — architektura, security config, warstwy — gotowe od razu
  • Boilerplate — DTOs, CRUD, mapowania, walidacje — generowane bezbłędnie
  • Dokumentacja — CLAUDE.md, README, Swagger, Postman — na bieżąco

Na co uważać


  • Code review obowiązkowy — kod nie przeszedłby produkcyjnego review bez poprawek
  • Reużywalność — komponenty bywają za duże (modals ~1100 linii), mało DRY
  • Mock data — AI lubi wrzucać hardcoded dane zamiast integrować z API
  • Konwersja typów — status "Aktywny" vs "ACTIVE" — trzeba pilnować spójności
  • Trzeba pilnować — konwencje, nazewnictwo, struktura plików wymagają stałego nadzoru
  • Kontekst — im lepszy CLAUDE.md, tym lepsze wyniki. Inwestycja w prompt się zwraca

Podsumowanie


AI drastycznie przyspiesza development — szczególnie scaffolding, CRUD, integrację i powtarzalny boilerplate. Zmiana bazy danych z Couchbase na MongoDB przeszła praktycznie bezproblemowo. Natomiast kod wymaga ludzkiego code review — AI generuje rozwiązania które działają, ale nie zawsze są optymalne pod kątem reużywalności i czystości. Kluczem jest dobra konfiguracja (CLAUDE.md) i traktowanie AI jako junior developera który jest bardzo szybki, ale wymaga nadzoru.
